## Multilingual Strategy (English / Japanese / Korean)

This portfolio should explicitly support three languages:
- **English** as the default and canonical source of truth for global recruiters, hiring managers, and collaborators
- **Japanese** for Japan-based opportunities, local hiring contexts, and cross-cultural communication
- **Korean** for domestic professional credibility, original nuance, and source fidelity

### Recommended policy
- Write the **master content in English first**.
- Create **carefully localized Japanese and Korean versions**, not raw machine-translated duplicates.
- Keep structure mostly identical across languages, but allow selective adaptation for tone and expectations.

### Why this matters
- The user is targeting a globally legible product profile while also preparing for Japan-based opportunities.
- English supports broad discoverability and link-sharing.
- Japanese supports trust and accessibility for recruiters and interviewers in Japan.
- Korean preserves precision for source materials and local sharing.

### Recommended implementation for v1
Use internationalized routing:
- `/` -> English homepage
- `/ja` -> Japanese homepage
- `/ko` -> Korean homepage

Or alternatively:
- `/en`, `/ja`, `/ko`

For simplicity and SEO clarity, using `/en`, `/ja`, `/ko` is recommended.

### Content model
Each major content block should support three localized fields:
- `title`
- `summary`
- `body`
- `ctaLabel`

Use a structured content layer such as JSON, YAML, or TypeScript objects so Codex can generate pages from a single content system.

### Translation policy by section
**Translate fully:**
- Hero
- About
- Core strengths
- Work / case study summaries
- Resume snapshot
- Contact / CTA

**Translate selectively or partially:**
- Deep case study details with internal sensitivity
- Writing archive excerpts
- Older project notes

### Tone guidance by language
**English**
- Crisp, global, product-focused
- Slightly strategic and execution-oriented
- Avoid buzzword overload

**Japanese**
- Slightly more modest and trustworthy
- Emphasize collaboration, clarity, and continuous improvement
- Avoid sounding too self-promotional

**Korean**
- Professional, precise, and grounded
- Preserve nuance from source experience and planning work
- Can be slightly denser than English if needed

### Navigation recommendation
Add a visible language switcher in the header:
- EN | 日本語 | 한국어

Behavior:
- Preserve current page context when switching language where possible
- Fallback to English if a localized page is missing

### Practical v1 scope recommendation
For the first launch, fully localize these pages:
- Home
- About
- Work (top-level summaries only)
- Resume summary
- Contact

Then expand later to:
- Full case studies
- Writing archive
- Detailed project pages

### Codex instruction
The site must be built from the start with multilingual architecture, even if some secondary pages initially ship only in English.
English remains the canonical default, but Japanese and Korean should be first-class citizens in routing, navigation, and content structure.
